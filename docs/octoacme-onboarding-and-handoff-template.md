# OctoAcme — Onboarding & Handoff Template

## Purpose
Ensure smooth onboarding of new team members and effective handoffs between personas to maintain project continuity, knowledge transfer, and collaboration quality.

---

## New Team Member Onboarding Checklist

### General Onboarding (All Roles)
- [ ] Access granted to necessary systems and tools (GitHub, project boards, communication channels)
- [ ] Introduction to team members and key stakeholders
- [ ] Review of project overview and current status
- [ ] Access to project documentation (docs/, README, runbooks)
- [ ] Understanding of team rhythm (standups, planning, retrospectives)
- [ ] Review of communication norms and escalation paths

### Role-Specific Onboarding

#### Developer Onboarding
- [ ] Development environment setup completed
- [ ] Codebase walkthrough and architecture overview
- [ ] Review of coding standards and PR workflow
- [ ] Access to CI/CD pipelines and deployment processes
- [ ] Pair programming session with senior developer
- [ ] First starter task assigned and completed

#### Product Manager Onboarding
- [ ] Product vision and roadmap review
- [ ] Customer personas and user research insights shared
- [ ] Review of prioritization framework and decision-making process
- [ ] Introduction to key stakeholders and sponsors
- [ ] Understanding of success metrics and KPIs
- [ ] Shadow product planning and stakeholder meetings

#### Project Manager Onboarding
- [ ] Project plans, timelines, and milestones review
- [ ] Risk register and dependency tracking walkthrough
- [ ] Understanding of reporting cadence and formats
- [ ] Introduction to cross-team coordination touchpoints
- [ ] Review of escalation procedures
- [ ] Shadow key project ceremonies (planning, status meetings)

#### UX Designer Onboarding
- [ ] Design system and brand guidelines review
- [ ] User research findings and personas shared
- [ ] Access to design tools (Figma, Sketch, etc.)
- [ ] Review of accessibility standards and requirements
- [ ] Understanding of design review process and feedback cycles
- [ ] Introduction to Product Manager and Development team

#### DevOps Engineer Onboarding
- [ ] Infrastructure architecture and IaC overview
- [ ] Access to cloud platforms and deployment tools
- [ ] Review of CI/CD pipeline configuration
- [ ] Monitoring, logging, and alerting systems walkthrough
- [ ] On-call rotation and incident response procedures
- [ ] Review of deployment schedules and maintenance windows

#### Security Lead Onboarding
- [ ] Security policies and compliance requirements review
- [ ] Threat model and risk assessment documentation
- [ ] Access to security scanning tools and SIEM systems
- [ ] Review of incident response playbook
- [ ] Understanding of vulnerability management process
- [ ] Introduction to security review and approval workflows

#### Data Analyst Onboarding
- [ ] Data infrastructure and analytics platform access
- [ ] Review of existing dashboards and reports
- [ ] Understanding of metrics definitions and KPIs
- [ ] Data quality standards and validation processes
- [ ] Introduction to data pipeline and ETL workflows
- [ ] Review of experimentation framework and A/B testing approach

#### Sponsor/Executive Stakeholder Onboarding
- [ ] Strategic objectives and business context review
- [ ] High-level project roadmap and milestones
- [ ] Understanding of escalation protocols
- [ ] Review of executive reporting format and cadence
- [ ] Key decisions requiring sponsor approval identified
- [ ] Introduction to project leadership team

## Cross-Persona Collaboration Guidelines

### Effective Handoffs Between Roles

#### Product Manager → UX Designer
- Share user research, problem statements, and acceptance criteria
- Clarify constraints (timeline, technical, business)
- Define success metrics for UX improvements
- Schedule design review checkpoints

#### UX Designer → Developer
- Provide design specs, prototypes, and interaction guidelines
- Clarify accessibility requirements and standards
- Be available for questions during implementation
- Schedule design review of implementation

#### Developer → QA
- Document test scenarios and edge cases
- Provide local testing instructions
- Highlight risk areas or complex changes
- Be available for bug triage and questions

#### QA → DevOps Engineer
- Share test results and coverage reports
- Identify smoke test scenarios for deployment validation
- Communicate any environment-specific issues
- Coordinate on deployment verification

#### DevOps Engineer → Security Lead
- Share infrastructure changes and their security implications
- Provide access to security scanning results
- Coordinate on vulnerability remediation
- Collaborate on incident response

#### Security Lead → Project Manager
- Communicate security requirements early in planning
- Provide risk assessments for project timeline
- Escalate critical security issues promptly
- Share security review timelines and requirements

#### Data Analyst → Product Manager
- Present insights with actionable recommendations
- Clarify data limitations and confidence levels
- Validate metric definitions and calculations
- Collaborate on experiment design

#### Project Manager → Sponsor/Executive Stakeholder
- Provide concise status updates with business context
- Escalate decisions requiring executive approval
- Communicate risks with impact and mitigation plans
- Request resources or unblock organizational issues

### Communication Best Practices
- **Be proactive:** Don't wait for problems to escalate
- **Provide context:** Explain the "why" behind requests or decisions
- **Set clear expectations:** Define timelines, deliverables, and success criteria
- **Document decisions:** Keep a record of key choices and their rationale
- **Follow up:** Ensure handoffs are complete and understood
- **Ask questions:** Seek clarification when unclear

## Offboarding Checklist

When a team member leaves the project:
- [ ] Complete handoff template with successor or team
- [ ] Document tribal knowledge in project docs
- [ ] Transfer ownership of ongoing work items
- [ ] Update RACI or responsibility matrix
- [ ] Conduct exit interview to capture lessons learned
- [ ] Remove access to systems (if leaving organization)
- [ ] Update team contact lists and org charts
